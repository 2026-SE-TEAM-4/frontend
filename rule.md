# 코딩 규칙

본 레포는 한국공학대학교 소프트웨어공학 (01) 4조 학기 프로젝트의 일부이다.
백엔드·서버풀 레포와 동일한 5개 원칙을 따르며, 아래는 프론트엔드 적용 사항이다.

## 1. 곁가지 기능을 만들지 않는다

- 유스케이스 명세에 있는 화면만 만든다. 기초공사 단계에서는 연결 상태 표시까지만.
- 라이브러리(React·Tailwind·shadcn)가 주는 기능을 직접 다시 만들지 않는다.

## 2. 학부생 수준에서 이해 가능해야 한다

- 한 컴포넌트의 책임이 한눈에 들어와야 한다. 커지면 분리한다.
- 과한 추상화·제너릭·HOC 남용을 피한다.

## 3. 코드는 자기 설명적이어야 한다

- 이름이 곧 설명이다. 주석은 *왜*를 적고, 이모지를 넣지 않는다.

## 4. 학술 자료로서의 일관성

- 명세서·다이어그램·코드의 용어를 일치시킨다(예약/Reservation 혼용 금지).

## 5. 변경 절차

- 디렉토리 구조를 바꾸려면 `tree.md` 를 먼저 갱신한다.
- 외부 계약(백엔드 API 응답 형태 등)에 의존하는 코드는 설계 문서를 단일 출처로 삼는다.

## 프론트엔드 적용

- 스타일은 Tailwind 유틸리티를 기본으로, 디자인 토큰(CSS 변수)을 따른다.
- 색·폰트·간격을 컴포넌트마다 하드코딩하지 않는다(토큰 사용).
- 데이터 호출은 상위/훅에서, 표현은 하위 컴포넌트에서.
